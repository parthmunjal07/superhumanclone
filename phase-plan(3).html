Security
Todo
Move access token out of localStorage critical
localStorage is readable by any script on the page — a single XSS hole leaks every token. Store the access token in memory (React context / Zustand) and the refresh token in an httpOnly; Secure; SameSite=Strict cookie. On page load, hit /api/auth/refresh to rehydrate silently.
securityfrontendbackend
Todo
Helmet + CSRF protection critical
Add helmet() middleware to set secure HTTP headers (CSP, HSTS, X-Frame-Options, etc.). Add CSRF tokens on all state-mutating endpoints using csurf or the Double Submit Cookie pattern. Validate Origin / Referer headers as a secondary check.
securitybackend
Todo
Rate limiting on auth endpoints
Use express-rate-limit on /api/auth/login, /api/auth/signup, and /api/auth/refresh. Recommended limits: 10 req/15 min per IP on login/signup, 30 req/15 min on refresh. Return 429 with a Retry-After header.
securitybackend
Todo
Request body size limits
Set express.json({ limit: '50kb' }) globally. Override to '5mb' only on the compose/send endpoint that accepts attachments. Prevents memory exhaustion from oversized payloads.
securitybackend
Architecture
Todo
Service layer — separate controllers from business logic
Controllers should only parse the request and call a service. Services own DB queries, validation, and business logic. Example: EmailController.send() calls EmailService.send() which calls CorsairClient.sendEmail(). Never write a Prisma query inside a route handler.
architecturebackend
Todo
Consistent JSON response shape
Every endpoint returns the same envelope: {"success": true, "data": {...}} or {"success": false, "error": {"code": "NOT_FOUND", "message": "..."}}. Create a sendSuccess() and sendError() helper. No one-off shapes anywhere.
architecturebackend
Todo
Graceful shutdown
Listen for SIGTERM and SIGINT. On signal: stop accepting new requests, drain in-flight requests (with a 10s timeout), close Prisma connection, close Redis client, then exit with code 0. Prevents data corruption on Vercel redeploys.
